# Програвач касет (автомобільний)

## Загальна характеристика

Програвач касет — це інтерактивний елемент інтер’єру транспортного засобу, який дозволяє водієві та пасажирам відтворювати аудіозаписи зі спеціальних предметів-касет, що знаходяться в рюкзаку гравця. Система забезпечує синхронізоване відтворення для всіх гравців поблизу автомобіля, створюючи ефект реальної аудіосистеми.

Програвач інтегровано в панель приладів (або іншу частину салону) і керується через клавіші, а візуальний зворотний зв’язок відображається на спеціальному GUI-інтерфейсі, який можна приховувати для зручності.

## Призначення

Основна мета програвача — створення атмосфери та можливість персоналізації поїздок шляхом відтворення музичних композицій або звукових ефектів. Система дозволяє:

- обирати касети з інвентаря;
- керувати відтворенням (відтворення, пауза, зупинка, перемикання треків);
- синхронізувати звук для всіх гравців у радіусі чутності;
- бачити поточний час відтворення та назву композиції.

## Ігровий процес

Для використання програвача гравець повинен:

1. Сісти за кермо транспортного засобу, обладнаного програвачем касет.
2. Мати в рюкзаку хоча б один предмет типу `Cassette` (касета) з правильно налаштованими атрибутами (ID, Name, Duration).
3. Відкрити інтерфейс програвача (за замовчуванням він прихований) і використовувати клавіші керування.

Інтерфейс автоматично сканує рюкзак гравця та формує список доступних касет. Перша касета в списку (завжди присутня `None`) означає відсутність активної касети.

Гравець може:

- перемикати касети вперед/назад;
- запускати відтворення вибраної касети;
- призупиняти або зупиняти програвач;
- приховувати або показувати GUI.

Усі дії відбуваються в реальному часі, а звук чути всім гравцям у межах зони чутності (радіус 120 одиниць). Якщо гравець виходить із цієї зони, він перестає чути музику, але синхронізація часу відновлюється при повторному входженні.

## Механіка роботи

Програвач складається з двох основних частин:

- **Клієнтська частина (GUI та управління)** — відповідає за відображення інтерфейсу, обробку натискань клавіш, комунікацію з сервером через `RemoteEvent`.
- **Серверна частина (відтворення звуку та синхронізація)** — керує звуковим об'єктом, транслює стан відтворення всім активним клієнтам.

### Клієнтська логіка

1. При завантаженні програвач сканує рюкзак гравця, знаходячи всі інструменти з назвою, що починається з `Cassette`. Для кожної касети зчитується конфігурація (`Configuration`), з якої отримуються атрибути:
   - `ID` — ідентифікатор звуку (SoundId) для відтворення;
   - `Name` — назва композиції (відображається в інтерфейсі);
   - тривалість (`Duration`) визначається шляхом завантаження тимчасового `Sound` та читання його властивості `TimeLength`.
2. Формується список касет, першим елементом завжди йде пуста касета (`None`) для скидання вибору.
3. Інтерфейс оновлюється відповідно до поточної касети: назва, прогрес-бар, поточний час, загальна тривалість.
4. Клавіші керування надсилають відповідні дії на сервер через `remote:FireServer(action, data)`.
5. Сервер періодично надсилає клієнту оновлення часу (`TIME`) для синхронізації прогрес-бару, навіть коли програвач призупинено.

### Серверна логіка

1. При старті створюється звуковий об'єкт (`Sound`) у складі автомобіля з налаштуваннями:
   - `RollOffMode = LinearSquare` (згасання з відстанню);
   - `EmitterSize = 15` (розмір джерела);
   - `MaxDistance = 120` (максимальна дистанція чутності);
   - `Looped = false` (не зациклюється автоматично).
2. Сервер отримує дії від клієнтів:
   - `PLAY` — встановлює `SoundId`, `TimePosition` і запускає відтворення.
   - `PAUSE` — призупиняє звук.
   - `RESUME` — відновлює з переданої позиції.
   - `STOP` — зупиняє та скидає позицію.
3. Активні клієнти відстежуються за допомогою пінгів (`PING`), які клієнт надсилає кожні 0.5 секунди. Якщо пінг не надходить більше 2 секунд, клієнт вважається неактивним.
4. Кожні 0.1 секунди (на `Heartbeat`) сервер надсилає поточний час відтворення `sound.TimePosition` усім активним клієнтам, які перебувають у радіусі чутності (`MaxDistance`), забезпечуючи синхронізацію.

### Синхронізація

Оскільки звук відтворюється виключно на сервері (у межах автомобіля), усі клієнти отримують однаковий аудіопотік. Проте GUI (прогрес-бар) має відображати поточний час незалежно. Для цього сервер розсилає оновлення часу, а клієнт оновлює свій інтерфейс. Це дозволяє уникнути розсинхронізації через затримки мережі.

## Керування

Управління програвачем здійснюється за допомогою клавіш, коли гравець перебуває в автомобілі. Для роботи клавіш не потрібно фокусувати GUI — вони діють глобально.

| Клавіша | Дія |
|---------|-----|
| `P`     | Відтворити поточну касету (якщо не відтворюється) |
| `U`     | Призупинити відтворення |
| `R`     | Відновити відтворення (з поточної позиції) |
| `T`     | Зупинити відтворення (скинути позицію до 0) |
| `[`     | Перемкнути на попередню касету в списку |
| `]`     | Перемкнути на наступну касету в списку |
| (затиснути `0`) + `P` | Показати GUI програвача |
| (затиснути `0`) + `U` | Приховати GUI програвача |

!!! note "Примітка"
    Клавіші `[` і `]` відповідають клавішам на фізичній клавіатурі (ліва та права квадратна дужка). Якщо використовується нестандартна розкладка, їх може бути змінено в коді.

## Інтерфейс

GUI програвача містить такі елементи (позначення відповідають назвам у скрипті):

- `Name` (`Title`) — рядок із назвою поточної касети (композиції).
- `ProgressBar` (`ProcessBar`) — горизонтальна шкала, що заповнюється пропорційно до поточного часу відтворення.
- `Time` — поточний час відтворення у форматі `M:SS`.
- `MaxTime` — загальна тривалість касети у форматі `M:SS`.

Інтерфейс автоматично приховується через кілька секунд після останньої дії (або за командою) і знову з’являється за натисканням спеціальної комбінації. Позиція GUI — у нижній частині екрана; він плавно виїжджає/з’їжджає за допомогою анімацій `TweenService`.

## Касети

Касети — це предмети категорії `Tool`, які повинні мати дочірній об'єкт `Configuration` із заданими атрибутами:

- `ID` (рядок) — посилання на аудіофайл (SoundId) у вигляді `rbxassetid://...` або повного URL.
- `Name` (рядок) — назва треку, що відображатиметься в інтерфейсі.

Тривалість визначається автоматично під час сканування, тому вказувати її в атрибутах не потрібно.

!!! tip "Порада"
    Щоб касета була правильно розпізнана, її назва має починатися зі слова `Cassette` (наприклад, `CassetteRock`, `CassetteJazz`). Це жорстко закодовано в скрипті.

Якщо касета не має атрибута `ID` або звук не завантажується, вона додається до списку з тривалістю 0, але відтворення не буде запущено.

## Технічні деталі

- **Звуковий рушій** — використовується стандартний `Sound` з властивостями для 3D-позиціонування.
- **Синхронізація** — оновлення часу надсилаються з частотою 10 Гц (0.1 с) лише для гравців у зоні чутності.
- **Пінг-механізм** — підтримує список активних слухачів, автоматично очищаючи їх після 2 секунд бездіяльності.
- **Анімація GUI** — використовується `TweenService` для плавного появи/зникнення.
