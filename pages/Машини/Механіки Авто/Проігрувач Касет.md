# Касетний плеєр у транспорті (Car Cassette Player System)

**Категорія:** Системи транспорту / Музичні системи  
**Тип:** Ігровий функціонал (Roblox / HoRP)  
**Оновлення:** Sync-based Cassette Audio System v2  

---

## Опис системи

Касетний плеєр у транспорті — це серверно-клієнтська система відтворення аудіо в автомобілях, яка дозволяє гравцю:

- завантажувати касети з інвентаря (Backpack)
- перемикати треки/касети
- керувати відтворенням (Play / Pause / Resume / Stop)
- синхронізувати час програвання між гравцями
- отримувати реалістичний звук із позиційним розповсюдженням у просторі

---

## Архітектура системи

Система складається з двох частин:

### 1. Клієнтська частина (LocalScript)
Відповідає за:
- UI інтерфейс касетного плеєра
- обробку клавіш
- вибір касет із інвентаря
- відправку команд на сервер
- відображення часу та прогресу

### 2. Серверна частина (Script)
Відповідає за:
- відтворення звуку в автомобілі
- синхронізацію часу
- контроль активних клієнтів
- перевірку дистанції гравця
- розсилку поточного часу відтворення

---

## Система касет

### Завантаження касет

Касети беруться з `Backpack` гравця:

- шукаються об’єкти `Tool`
- назва повинна починатися з `Cassette`
- всередині має бути `Configuration`
- атрибути:
  - `ID` — SoundId
  - `Name` — назва касети

---

### Отримання тривалості

Для кожної касети:
1. Створюється тимчасовий `Sound`
2. Призначається `SoundId`
3. Очікується завантаження
4. Береться `TimeLength`
5. Об’єкт видаляється

---

## UI система

Інтерфейс включає:

- **ProgressBar** — прогрес відтворення
- **Time** — поточний час
- **MaxTime** — загальна тривалість
- **Name/Title** — назва касети

### Анімації UI:
- плавне з’явлення / приховування
- Tween переміщення вниз/вгору
- зміна прозорості всіх елементів

## Керування

### Клавіші

| Клавіша | Дія |
|--------|-----|
| P | Play / Show GUI |
| U | Pause / Hide GUI |
| R | Resume |
| S | Stop |
| [ | Next cassette |
| ] | Previous cassette |

> Умовно UI активується лише коли плеєр відкритий

## Синхронізація (Client → Server)

### Ping система
Клієнт кожні 0.5 сек надсилає: remote:FireServer("PING")


Сервер використовує це для:
- визначення активних гравців
- видалення неактивних клієнтів

---

## Відтворення аудіо (Server Logic)

### Основні стани:

- `PLAY`
- `PAUSE`
- `RESUME`
- `STOP`

### Логіка:

- звук створюється на `Speakers` або `PrimaryPart`
- використовується `LinearSquare` затухання
- максимальна дистанція: `120 studs`
- emitter size: `15`

---

## Синхронізація часу

Сервер кожні ~0.1 сек:

1. перевіряє чи звук грає
2. обчислює позицію авто
3. перевіряє дистанцію гравців
4. відправляє: remote:FireClient(player, "TIME", sound.TimePosition)


---

## Формула прогресу

progress = currentTime / totalTime


Обмеження:
- clamp від 0 до 1
- використовується для UI прогрес-бару

---

## Перемикання касет

### Логіка:
- індексація списку `cassetteList`
- циклічний перехід:
  - next: `(index % count) + 1`
  - prev: `(index - 2) % count + 1`

---

## Безпека та стабільність

Система включає:
- перевірку існування `Car`
- перевірку `SoundId`
- обробку відсутніх касет
- очищення неактивних клієнтів
- перевірку дистанції перед sync

---

## Примітки

- Система не використовує глобальний streaming аудіо, тільки локальний Sound emitter
- Кожен гравець отримує синхронізований час, але звук генерується сервером
- UI не впливає на серверну логіку
